# 需求文档

## 1. 应用概述

**应用名称**：投屏助手

**应用描述**：一款移动端投屏接收软件，作为大屏端接收其他设备推送的视频内容并播放。支持扫码投屏和局域网投屏两种连接方式，能够播放所有视频编码格式，自动选择正确音轨并挂载简体中文字幕。

## 2. 用户与使用场景

**目标用户**：需要将手机、平板或电脑作为大屏播放设备，接收其他设备投屏内容的用户。

**核心使用场景**：
- 用户将手机/平板连接到电视或投影仪，打开本App作为投屏接收端，其他设备扫码或通过局域网连接后推送视频进行播放
- 用户在会议室将设备作为投屏接收端，接收其他参会者推送的视频内容

## 3. 页面结构与功能说明

### 3.1 页面结构

```
投屏助手App（接收端）
├── 待连接页面
│   ├── 二维码展示区域
│   ├── 局域网连接信息展示
│   └── 连接状态提示
├── 视频播放页面
│   ├── 视频播放区域
│   ├── 播放控制栏
│   ├── 播放列表显示
│   └── 字幕显示区域
└── 设置页面
    └── 外挂字幕加载
```

### 3.2 功能说明

#### 3.2.1 待连接页面

**功能描述**：
- App打开后默认进入待连接状态
- 页面中央展示二维码，供其他设备扫码连接
- 页面下方展示局域网连接信息（设备名称、IP地址等），供其他设备通过局域网搜索并连接
- 显示当前连接状态（待连接/已连接）
- 自动监听局域网内的投屏连接请求

**操作流程**：
- 用户打开App，页面自动生成二维码并展示局域网连接信息
- 其他设备扫描二维码或通过局域网搜索到本设备后发起连接
- 连接成功后，页面自动跳转至视频播放页面

#### 3.2.2 视频播放页面

**功能描述**：
- 接收发送端推送的视频内容并播放
- 显示视频播放画面
- 提供播放、暂停、进度调节、音量调节等基础控制功能
- 显示当前播放的音轨和字幕状态
- 支持播放列表模式，显示当前播放视频和列表中的其他视频
- 提供断开连接功能

**操作流程**：
- 发送端推送视频后，视频自动开始播放
- 用户通过控制栏操作视频播放（暂停、调节进度、调节音量）
- 播放列表模式下，当前视频播放完毕后自动播放下一个视频
- 用户点击断开连接按钮，停止播放并返回待连接页面

#### 3.2.3 设置页面

**功能描述**：
- 提供外挂字幕文件加载功能
- 用户可从本地选择字幕文件（SRT、ASS等格式）挂载到当前播放视频
- 发送端也可推送字幕文件到本App进行挂载

**操作流程**：
- 用户在视频播放页面点击设置按钮进入设置页面
- 用户点击「加载字幕」，从本地文件中选择字幕文件
- 字幕文件加载成功后，自动显示在视频播放画面中
- 发送端推送字幕文件时，自动加载并显示

## 4. 业务规则与逻辑

### 4.1 连接与投屏逻辑

- 扫码投屏和局域网投屏均需发送端与本App处于同一网络环境
- App打开后自动生成二维码并开启局域网广播，等待其他设备连接
- 发送端扫描二维码或通过局域网搜索到本设备后，发起连接请求
- 连接建立后，发送端推送视频到本App，本App自动开始播放
- 投屏过程中若网络中断，自动提示用户并尝试重新连接
- 用户主动断开连接后，停止播放并返回待连接页面

### 4.2 视频编码支持与自动转码规则

- 应用支持所有常见视频编码格式（H.264、H.265、VP9、AV1、MPEG-4等）
- 视频文件无论何种编码格式均可正常播放
- 当检测到视频编码格式不兼容或无法正常播放时，自动进行格式转码
- 转码过程在后台自动完成，用户无需任何操作
- 转码完成后视频自动开始播放

### 4.3 音轨自动选择规则

- 应用自动检测视频文件中的所有音轨
- 自动识别并过滤掉编码错误或无法播放的音轨
- 优先选择正常可播放的音轨进行输出
- 若存在多条正常音轨，优先选择中文音轨，其次为原始音轨

### 4.4 字幕自动挂载规则

- 应用自动检测视频文件中的内嵌字幕轨道
- 自动识别简体中文字幕轨道并挂载
- 若视频文件无内嵌简体中文字幕，则不显示字幕
- 用户加载外挂字幕文件后，外挂字幕优先于内嵌字幕显示
- 发送端推送字幕文件时，自动加载并优先显示
- 字幕挂载后自动同步显示在视频播放画面中

### 4.5 播放列表规则

- 发送端可推送多个视频组成播放列表
- 播放列表按发送端推送顺序播放
- 播放列表模式下，当前视频播放完毕后自动播放下一个视频
- 播放列表播放完毕后停止播放，等待发送端继续推送或断开连接

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 发送端扫码失败或二维码无效 | 提示「扫码失败，请重试」 |
| 局域网内无发送端连接 | 保持待连接状态，持续展示二维码和局域网信息 |
| 视频文件所有音轨均编码错误 | 提示「视频音轨异常，无法播放」 |
| 视频文件无简体中文字幕 | 不显示字幕，正常播放视频 |
| 投屏过程中网络中断 | 提示「网络连接中断，正在尝试重新连接」，尝试3次后提示「连接失败，请检查网络」 |
| 发送端推送的视频文件损坏或无法读取 | 提示「视频文件无法读取，请重新推送」 |
| 视频自动转码失败 | 提示「视频格式不兼容，无法播放」 |
| 外挂字幕文件格式不支持 | 提示「字幕文件格式不支持，请选择SRT或ASS格式」 |
| 播放列表播放过程中某个视频文件损坏 | 跳过该视频，自动播放下一个视频，并提示「视频文件异常，已跳过」 |
| 发送端推送空播放列表 | 提示「播放列表为空，请推送视频」 |

## 6. 验收标准

1. 用户打开App进入待连接页面，页面展示二维码和局域网连接信息
2. 发送端扫描二维码或通过局域网搜索到本设备，成功建立连接
3. 发送端推送一个视频文件到本App
4. 视频自动在本App上开始播放，自动选择正确音轨，自动挂载简体中文字幕
5. 用户点击暂停按钮，视频暂停播放
6. 用户点击断开连接，视频停止播放并返回待连接页面
